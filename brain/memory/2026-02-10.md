### Voice Call Fix
- Philip asked me to call him (showing off to parents who just arrived)
- First two attempts failed with "application error" — embarrassing
- Root cause: stale server processes fighting over port 8765
- First call used ConversationRelay TwiML (wrong protocol — server uses Media Streams)
- Second call used correct /twiml endpoint but hit the stale process
- Fix: kill all processes on 8765, clean restart, use server's own /twiml endpoint
- Third attempt worked — Gemini Live Native Audio flowing both ways
- Philip confirmed "it's live and good"

**Working startup procedure:**
```bash
# Kill any existing processes
fuser -k 8765/tcp
sleep 2

# Clean start
cd /home/philip/robothor/brain/voice-server
> server.log  # truncate log
PYTHONUNBUFFERED=1 \
  GOOGLE_API_KEY="AIzaSyDF8SSIEuykiiJ81R2T04FKrzyYLNhKg1o" \
  OPENAI_API_KEY="$OPENAI_API_KEY" \
  nohup ../voice-server-env/bin/python server.py >> server.log 2>&1 &

# Make call using server's TwiML endpoint (NOT ConversationRelay)
curl -X POST "https://api.twilio.com/2010-04-01/Accounts/$TWILIO_SID/Calls.json" \
  -u "$TWILIO_SID:$TWILIO_AUTH" \
  -d "To=+13479061511" \
  -d "From=+14134086025" \
  -d "Url=https://voice.robothor.ai/twiml"
```

**Key lesson:** NEVER use ConversationRelay TwiML with this server. Always use `Url=https://voice.robothor.ai/twiml` which returns the proper `<Stream>` TwiML.
